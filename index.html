Clean Architecture Boundaries

Frametric is structured as a Modular Monolith applying Clean Architecture patterns to prevent domain leakage. The business core remains completely decoupled from database engine models, third-party libraries, and file-parsing schemas:

1. Core Domain

Pure C# aggregates containing business rules and zero external imports.

2. Application Layer

Mediator CQRS handlers and interface abstractions.

3. Infrastructure

Database context persistent configurations, custom CsvHelper stream mappers, and external client integrations via the TMDB API.

4. API Layer

Thin REST controllers securing requests via JWT policies and input-validating middleware.

Pautas de Clean Architecture

Frametric está estructurado como un Monolito Modular aplicando patrones de Clean Architecture para evitar fugas de dominio. El núcleo de negocio permanece completamente desacoplado de los modelos de base de datos, bibliotecas de terceros y esquemas de análisis de archivos:

1. Core Domain

Agregados puros en C# que contienen las reglas de negocio y cero dependencias externas.

2. Application Layer

Mediator CQRS y abstracciones de interfaces.

3. Infrastructure

Persistencia de base de datos, mapeo de archivos csv con CsvHelper en flujo de memoria y cliente de API TMDB.

4. API Layer

Controladores REST simplificados con políticas JWT y middleware para validaciones de esquemas.

Transactional & Analytical Hybrid Persistence

Mirroring industry best practices, the database layer combines Entity Framework Core and Dapper to optimize specific operations:

  • EF Core (Write Path): Handles standard CRUD transactions, identity state tracking, database migrations, and complex relational updates (e.g., movie genre linkages).
  • Dapper (Read/Analytics Path): Bypasses EF Core tracking overhead to execute highly optimized SQL queries (using CTEs and index parameters) for dashboard histograms and ranking aggregates.
  • Strict PG Casting: Enforces explicit type casting (e.g., `CAST(COUNT(...) AS INTEGER)`) to map PostgreSQL variables accurately to immutable C# primary constructors.

Persistencia híbrida analítica y transaccional

Reflejando las mejores prácticas de la industria, la capa de datos combina Entity Framework Core y Dapper para optimizar operaciones clave:

  • EF Core (escritura): Gestiona transacciones CRUD estándar, ciclo de identidad, migraciones y relaciones complejas (ej. asociación de géneros).
  • Dapper (lecturas/analítica): Evita la carga de seguimiento de EF Core para ejecutar consultas SQL directas y altamente optimizadas (con CTEs y optimización de índices) para gráficos y métricas del dashboard.
  • Casteo en Postgres: Implementa conversiones explícitas de tipos en las consultas (ej. `CAST(COUNT(...) AS INTEGER)`) para asegurar el mapeo correcto con constructores primarios de C# inmutables.

System.Threading.Channels Pipeline

Bulk ZIP imports contain bare titles and years. The enrichment of genres, runtime minutes, and poster assets is decoupled to ensure fast response times:

  1. In-Memory Parsing: CsvHelper reads ZIP archive streams directly in memory without writing files to local disk.
  2. Asynchronous Signal: The import handler writes a lightweight wake-up signal to a `System.Threading.Channels.Channel`.
  3. Background Worker: A hosted `BackgroundService` reads from the channel and processes pending movies in batches of 20, incorporating a 10-second throttling delay to respect TMDB API rate limit policies.

Flujo asíncrono con System.Threading.Channels

La importación masiva de archivos ZIP contiene solo títulos y años. El enriquecimiento de metadatos (géneros, duración y posters) se desacopla para acelerar la respuesta de la API:

  1. Lectura en memoria: CsvHelper procesa los flujos del archivo comprimido directamente en memoria sin escribir archivos temporales en disco.
  2. Señalización asíncrona: El manejador de importación escribe un aviso ligero de activación en un `System.Threading.Channels.Channel`.
  3. Trabajador en segundo plano: Un `BackgroundService` hosted lee las señales del canal y procesa los enriquecimientos en lotes de 20 películas, respetando los límites de TMDB con pausas de 10 segundos.

Continuous Quality Assurance

To guarantee long-term maintenance and zero regressions, Frametric utilizes a multi-tiered automated testing framework:

  • Backend Unit Tests (xUnit & Moq): Covers command validation handlers, ZIP structure anomalies (e.g., converting decimals to nullable years), and database persistence triggers.
  • Frontend E2E Tests (Playwright): 45 automated test scenarios mapping routing guards, authorization redirects, stats query tables, and slideshow gesture keys across WebKit, Firefox, and Chromium.
  • Screenshot Generation: Playwright-based custom automation to capture all states of the visual presentation layers directly inside headless browser environments.

Aseguramiento de calidad continuo

Para garantizar la mantenibilidad y prevenir regresiones, Frametric cuenta con un robusto sistema de pruebas automatizadas en múltiples niveles:

  • Pruebas unitarias de backend (xUnit y Moq): Validaciones de comandos, gestión de inconsistencias en los ZIPs importados y lógica de guardado y persistencia en base de datos.
  • Pruebas E2E en frontend (Playwright): 45 escenarios interactivos simulando flujos reales de usuario, enrutamiento seguro, autenticación JWT, pantallas de estadísticas y vistas animadas en Chromium, WebKit y Firefox.
  • Generación de capturas: Automatización personalizada con Playwright para capturar todos los estados visuales del proyecto en navegadores sin encabezados.

Gamified Film Discovery & Rich State UI

To keep user engagement high, Frametric implements a suite of interactive game mechanics designed with micro-animations and robust client-side state management:

  • Cinematic Bingo: A grid-based objective board generated dynamically based on chosen scope. Checking local watch history dynamically marks grid intersections.
  • Selection Roulette: A high-performance CSS and JS wheel that animates sequentially through multiple movies until settling on a single winner.
  • Mystery Box: An interactive element where users click covered options to animate opening and revealing a hidden recommendation.
  • Cinematic Dice & Slots: Custom RNG rollers that match specific movie attributes (genres, decades, rating thresholds) to produce themed recommendations.

Descubrimiento gamificado e interfaces interactivas

Para maximizar el compromiso del usuario, Frametric implementa una suite de mecánicas de juego interactivas con microanimaciones y una sólida gestión de estado:

  • Bingo cinemático: Un tablero de objetivos interactivo generado dinámicamente según el alcance seleccionado. El historial de visualización marca de manera interactiva las casillas completadas.
  • Ruleta de selección: Una ruleta con transiciones CSS de alta fidelidad que se desplaza secuencialmente por diversas películas antes de declarar una ganadora.
  • Cajas misteriosas: Panel de cartas misteriosas interactivas donde el usuario hace clic para desencadenar animaciones de revelado de películas sugeridas.
  • Dados y tragaperras: Rodillos y generadores aleatorios que combinan variables de películas (décadas, géneros y valoraciones) para sugerir títulos bajo restricciones lógicas.
